# CloudPrintd v1.0.0 - Initial Release

## What's New

This is the first production-ready release of CloudPrintd, a self-hosted print server for connecting Salesforce cloud to on-site printers.

### Key Features

**Print Management:**
- Direct integration with Zebra ZPL thermal printers
- CUPS support for standard office printers
- Network printer auto-discovery
- Real-time printer status monitoring
- Print job statistics and tracking

**Easy Setup:**
- Interactive web-based setup wizard
- 5-minute installation process
- Automatic printer discovery
- Step-by-step configuration

**Connectivity Options:**
- Cloudflare Tunnel (no port forwarding needed)
- Tailscale VPN
- Dynamic DNS + Port Forwarding
- Local network mode

**Security:**
- Bearer token authentication
- Optional IP whitelisting
- Secure configuration management
- Systemd security hardening

**Update Management:**
- Package-based update system
- Zero-downtime version switching
- Automatic rollback on failure
- Version upgrade/downgrade capability
- Web-based update dashboard

**API Integration:**
- RESTful API for Salesforce integration
- Comprehensive API documentation
- Example Apex code provided
- Batch printing support

### System Requirements

- Raspberry Pi Zero 2W (minimum) or Raspberry Pi 4 (recommended)
- Raspberry Pi OS Lite
- Python 3.11+
- 512MB RAM minimum
- 2GB free disk space
- Network connectivity

### Installation

See [Setup Guide](../docs/setup-guide.md) for complete installation instructions.

Quick start:
```bash
# Copy files to /opt/CloudPrintd
sudo systemctl enable CloudPrintd
sudo systemctl start CloudPrintd

# Access setup wizard at:
http://<your-pi-ip>:8000/setup
```

### API Usage

```bash
# Submit print job
curl -X POST http://your-server:8000/api/v1/print \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "printer": "warehouse_zebra",
    "content": "^XA^FO50,50^A0N,50,50^FDTest^FS^XZ",
    "format": "zpl"
  }'
```

### Known Issues

- CUPS integration requires manual installation on minimal systems
- Network discovery limited to /24 subnets
- Maximum recommended throughput: 100 jobs/minute
